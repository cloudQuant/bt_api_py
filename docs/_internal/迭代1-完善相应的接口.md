## 1. 项目现状概述

bt_api_py 是一个统一多交易所交易 API 框架，已完成以下架构演进（迭代 2~6）：

- **ExchangeRegistry 注册表模式**（`bt_api_py/registry.py`）：交易所即插即用，新交易所只需注册 feed_class / exchange_data_class / balance_handler
- **exchange_registers 自动发现**（`bt_api_py/exchange_registers/`）：`bt_api.py` 通过 `pkgutil.iter_modules` 自动扫描，无需手动 import
- **统一错误框架**（`bt_api_py/error.py` + `bt_api_py/errors/`）：`UnifiedError` 继承 `BtApiError`，各交易所翻译器已拆分到 `errors/` 子模块
- **Capability 机制**（`bt_api_py/feeds/capability.py`）：每个 Feed 通过 `_capabilities()` 声明支持的功能集合
- **余额处理器**（`bt_api_py/balance_utils.py`）：`simple_balance_handler` / `nested_balance_handler` 通用函数

## 2. 文件结构规范

每个交易所涉及的文件（以 `xxx` 代替交易所名）：

```
bt_api_py/
├── configs/xxx.yaml                          # YAML 配置：base_url、rest_paths、wss_paths、kline_periods
├── containers/
│   └── exchanges/xxx_exchange_data.py        # 交易所数据类：URL、路径、符号转换（继承 ExchangeData）
├── errors/
│   └── error_framework_xxx.py               # [可选] 交易所错误翻译器（继承 ErrorTranslator）
├── exchange_registers/
│   └── register_xxx.py                       # ★ 注册模块：注册 feed_class、exchange_data_class、balance_handler
├── feeds/
│   └── live_xxx/
│       ├── __init__.py
│       ├── request_base.py                   # 请求基类：request()、async_request()、_get_xxx()、签名、认证、规范化函数
│       └── spot.py                           # Spot 子类：三层方法（get_xxx / async_get_xxx）
tests/
└── feeds/
    └── test_xxx.py                           # 测试文件
```

***与旧版文档的关键差异**：
- **注册文件从 `feeds/` 移至 `bt_api_py/exchange_registers/`**（迭代 2/3）
- **容器类（ticker/orderbook/bar/order）大多数交易所不需要单独创建**，在 `request_base.py` 的 normalize_function 中直接返回 dict 即可
- **错误翻译器是可选的**，仅主流交易所需要

## 3. 标准接口规范

### 3.1 接口列表

#### 公共行情接口（无需 API Key）

| 方法 | 同步 | 异步 | 说明 | 优先级 |
|------|------|------|------|--------|
| `get_tick(symbol)` | ✅ | `async_get_tick` | 获取最新行情 | **必须** |
| `get_depth(symbol, count)` | ✅ | `async_get_depth` | 获取深度/订单簿 | **必须** |
| `get_kline(symbol, period, count)` | ✅ | `async_get_kline` | 获取 K 线数据 | **必须** |
| `get_server_time()` | ✅ | `async_get_server_time` | 获取服务器时间 | 推荐 |
| `get_exchange_info()` | ✅ | `async_get_exchange_info` | 获取交易所信息 | 推荐 |
| `get_trade_history(symbol)` | ✅ | `async_get_trade_history` | 获取成交历史 | 推荐 |

| 方法 | 同步 | 异步 | 说明 | 优先级 |
|------|------|------|------|--------|

| `get_balance()` | ✅ | `async_get_balance` | 获取余额 | **必须** |

### 3.2 三层方法模式

每个接口遵循 **三层方法** 模式：

```python
# 第一层：内部方法（在 request_base.py 中）
def _get_tick(self, symbol, extra_data=None, **kwargs):
    """只负责组装请求参数，返回 (path, params, extra_data) 三元组"""
    path = self._params.get_rest_path("get_tick").replace("{symbol}", ...)
    params = {}
    extra_data = extra_data or {}
    extra_data.update({
        "request_type": "get_tick",
        "symbol_name": symbol,               # ★ 必须包含
        "asset_type": self.asset_type,
        "exchange_name": self.exchange_name,
        "normalize_function": self._get_tick_normalize_function,
    })
    return path, params, extra_data

# 第二层：同步方法（在 spot.py 中）
def get_tick(self, symbol, extra_data=None, **kwargs):
    path, params, ed = self._get_tick(symbol, extra_data, **kwargs)
    return self.request(path, params, extra_data=ed)

# 第三层：异步方法（在 spot.py 中）
def async_get_tick(self, symbol, extra_data=None, **kwargs):
    path, params, ed = self._get_tick(symbol, extra_data, **kwargs)
    self.submit(self.async_request(path, params, extra_data=ed), callback=self.async_callback)
```

### 3.4 规范化函数

每个 `_get_xxx_normalize_function` 必须返回 `(data_list, status)` 元组：

```python
@staticmethod
def _get_tick_normalize_function(input_data, extra_data):
    if not input_data:
        return [], False
    # 检查 API 错误（根据交易所响应格式判断）
    if isinstance(input_data, dict) and "errors" in input_data:
        return [], False
    # 解包数据（根据交易所响应格式）
    data = input_data.get("data", input_data)
    if not data:
        return [], False
    return [data], True
```

### 步骤 2：创建 ExchangeData 类

创建 `bt_api_py/containers/exchanges/xxx_exchange_data.py`，参照 `bigone_exchange_data.py`：

```python
import os
from bt_api_py.containers.exchanges.exchange_data import ExchangeData
from bt_api_py.logging_factory import get_logger

logger = get_logger("xxx_exchange_data")
_xxx_config = None
_xxx_config_loaded = False

def _get_xxx_config():
    global _xxx_config, _xxx_config_loaded
    if _xxx_config_loaded:
        return _xxx_config
    try:
        from bt_api_py.config_loader import load_exchange_config
        config_path = os.path.join(
            os.path.dirname(os.path.dirname(os.path.dirname(os.path.abspath(__file__)))),
            "configs", "xxx.yaml",
        )
        if os.path.exists(config_path):
            _xxx_config = load_exchange_config(config_path)
        _xxx_config_loaded = True
    except Exception as e:
        logger.warn(f"Failed to load xxx.yaml: {e}")
    return _xxx_config

class XxxExchangeData(ExchangeData):
    def __init__(self):
        super().__init__()
        self.exchange_name = "XXX___SPOT"
        self.rest_url = "https://api.xxx.com"
        self.wss_url = "wss://ws.xxx.com"
        self.rest_paths = {}
        self.kline_periods = {"1m": "1min", "1h": "60min", "1d": "1day"}
        self.legal_currency = ["USDT", "USD", "BTC"]

    def _load_from_config(self, asset_type):
        """从 YAML 加载配置，失败时使用硬编码默认值"""
        config = _get_xxx_config()
        if config is None:
            return False
        asset_cfg = config.asset_types.get(asset_type)
        if asset_cfg is None:
            return False
        # 加载逻辑与 bigone_exchange_data.py 相同
        if asset_cfg.exchange_name:
            self.exchange_name = asset_cfg.exchange_name
        if hasattr(asset_cfg, "rest_url") and asset_cfg.rest_url:
            self.rest_url = asset_cfg.rest_url
        elif config.base_urls and config.base_urls.rest:
            rest = config.base_urls.rest
            self.rest_url = (rest.get(asset_type, rest.get("default", self.rest_url))
                            if isinstance(rest, dict) else rest)
        if asset_cfg.rest_paths:
            self.rest_paths.update(asset_cfg.rest_paths)
        kp = asset_cfg.kline_periods or (config.kline_periods if config.kline_periods else None)
        if kp:
            self.kline_periods = dict(kp)
        return True

    def get_symbol(self, symbol):
        """将标准 symbol 转为交易所格式"""
        return symbol.upper().replace("/", "").replace("-", "")

    def get_period(self, period):
        return self.kline_periods.get(period, period)

    def get_rest_path(self, request_type):
        path = self.rest_paths.get(request_type)
        if path is None:
            raise ValueError(f"Unknown rest path: {request_type}")
        return path

class XxxExchangeDataSpot(XxxExchangeData):
    def __init__(self):
        super().__init__()
        self.asset_type = "spot"
        if not self._load_from_config("spot"):
            self.rest_paths = {
                "get_server_time": "GET /api/v1/time",
                # ... 其余硬编码 fallback
            }
```

### 步骤 3：创建 request_base.py

创建 `bt_api_py/feeds/live_xxx/__init__.py`（空文件）和 `bt_api_py/feeds/live_xxx/request_base.py`。

参照 `live_bigone/request_base.py`，***必须实现的核心方法**：

1. **`__init__`** — 初始化参数、创建 ExchangeData 实例
2. **`_capabilities()`** — 声明支持的 Capability 集合
3. **`request()`** — 同步 HTTP 请求，返回 `RequestData`
4. **`async_request()`** — 异步 HTTP 请求，返回 `RequestData`
5. **`async_callback()`** — 将结果推入 data_queue
6. **`_get_xxx()` 系列** — 每个接口的参数组装方法
7. **`_get_xxx_normalize_function()` 系列** — 每个接口的数据规范化函数

***关键要点**：
- `request()` 签名：`def request(self, path, params=None, body=None, extra_data=None, timeout=10, is_sign=False)`
- path 格式统一为 `"METHOD /endpoint"`（如 `"GET /api/v1/ticker"`），在 request() 内 split 解析
- extra_data 中**必须**包含 `symbol_name`、`request_type`、`asset_type`、`exchange_name`、`normalize_function`

## 6. 常见问题与修复模式

### 6.1 源代码常见 bug

| 问题 | 典型错误 | 修复方式 |
|------|----------|----------|
| 缺少 `request()` 方法 | `AttributeError: no attribute 'request'` | 在 `request_base.py` 添加 `request()` + `async_request()` |
| `get_server_time()` 返回 tuple | `AssertionError: isinstance(tuple, RequestData)` | 改为调用 `self.request()` 返回 RequestData |
| `init_data()` 未返回 self | `AttributeError: 'NoneType' object...` | 在 `init_data()` 末尾 `return self` |
| 规范化函数缺少 symbol_name | `assert None == "BTC-USDT"` | 在 `_get_xxx()` 的 extra_data 中添加 `symbol_name` |
| `base_urls.rest` 是 dict | URL 拼接异常 | 从 dict 中按 asset_type 提取 URL 字符串 |
| REST 端点路径过时 | HTTP 404 | 查询最新 API 文档，更新 YAML/代码中的路径 |
| 方法名不匹配 | `get_tick` vs `get_ticker` | 添加别名 `get_ticker = get_tick` |
| 双重 request 调用 | 请求发送两次 | `_get_xxx` 只返回三元组，`get_xxx` 负责调用 `request()` |
| MRO 冲突 | `TypeError: Cannot create consistent MRO` | 删除多余的 Mixin 继承（已通过基类继承链获得） |
| 缩进/语法错误 | `IndentationError`/`SyntaxError` | auto-discover 会暴露所有 register 的导入链错误 |

### 6.2 测试常见 bug

| 问题 | 典型代码 | 修复方式 |
|------|----------|----------|
| mock 路径错误 | `patch("bt_api_py.feeds.feed.requests")` | 改为 `patch.object(XxxRequestDataSpot, "http_request", ...)` |
| `if...pass` 不保护后续代码 | `if data: pass; x = data[0]` | 改为 `if data: x = data[0]; assert ...` |
| 异步测试死代码 | `except Empty: pass; data = None; assert data` | 去掉 `data = None`，将 assert 放入 try 块 |
| 网络测试未标记 | 测试直接调用真实 API | 添加 `@pytest.mark.integration` |
| 容器 init_data 未调用 | `tick.get_last_price()` 返回 None | 先调用 `tick.init_data()` |
| 注册测试访问内部属性 | `ExchangeRegistry._feed_classes` | 改用 `ExchangeRegistry.has_exchange("XXX___SPOT")` |

## 7. 验收标准

1. 执行 `SKIP_LIVE_TESTS=true pytest tests/feeds/test_xxx.py -q --tb=short -o "addopts="` 结果为 **0 failed**
2. 公共行情接口（get_tick、get_depth、get_kline）有对应的同步 mock 测试且通过
3. 私有接口测试均标记 `@pytest.mark.integration`，SKIP_LIVE_TESTS=true 时跳过
4. 注册模块存在且自动注册生效（`ExchangeRegistry.has_exchange("XXX___SPOT")` 返回 True）
5. `_capabilities()` 正确声明支持的功能
6. 不允许通过删除断言或注释测试来"修复"失败
7. 完成后在 `README.md` 中更新支持列表

## 8. 限制与约束

1. ***参照 BigONE/Binance 标准**：所有新交易所实现应参考上述参照模板
2. ***一次只处理一个交易所**：完全完成后才能进入下一个
3. ***不允许一次运行全部测试**：耗时过长，每次只运行单个交易所的测试文件
4. ***源代码优先**：测试失败时先修源代码，再改测试
5. ***不可削弱测试**：不允许通过删除断言或降低覆盖率来"修复"
6. ***DEX 交易所后置**：DEX/DeFi 的接口模式与 CEX 差异大，优先处理 CEX
7. ***注册模块自动发现**：新交易所只需在 `exchange_registers/` 下创建 `register_xxx.py`，无需修改 `bt_api.py`
