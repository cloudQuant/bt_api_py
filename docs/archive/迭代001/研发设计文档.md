## 3. 插件契约规范 (正式契约)

### 3.1 entry_points group 名

```toml
# 插件包 pyproject.toml
[project.entry-points."bt_api_py.plugins"]
binance = "bt_api_binance.plugin:register_plugin"
```

核心加载器:

```python
from importlib.metadata import entry_points
for ep in entry_points(group="bt_api_py.plugins"):
    register_fn = ep.load()
    register_fn(registry, runtime)
```

### 3.2 PluginInfo 数据类

位置:`bt_api_py/plugins/protocol.py` (新增)

```python
from __future__ import annotations
from dataclasses import dataclass, field

@dataclass(frozen=True)
class PluginInfo:
    """每个插件必须返回的元信息."""
    name: str                        # 如 "bt_api_binance"
    version: str                     # 插件自身版本, 遵循 semver
    core_requires: str               # PEP 440 规格, 如 ">=2.0,<3.0"
    supported_exchanges: tuple[str, ...]   # 如 ("BINANCE___SPOT", "BINANCE___SWAP")
    supported_asset_types: tuple[str, ...] # 如 ("SPOT", "SWAP")
    plugin_module: str = ""          # 自动填充: ep.module
```

### 3.3 插件入口函数签名

每个插件的 `plugin.py` **必须** 暴露:

```python
def register_plugin(
    registry: "ExchangeRegistry",
    runtime_factory: "GatewayRuntimeRegistrar",
) -> PluginInfo:
    """Register this plugin's capabilities.

    - Called exactly once at core startup.
    - Must not raise unless registration truly cannot continue.
    - Partial registration is NOT allowed: if any step fails,
      raise PluginRegistrationError and let the core record it.
    - MUST return PluginInfo describing this plugin.
    """
```

### 3.4 `GatewayRuntimeRegistrar` 的职责

当前 `GatewayRuntime` 自身是实例化对象,不适合在启动阶段直接传给插件。引入一个**薄注册器**:

位置:`bt_api_py/gateway/registrar.py` (新增)

```python
class GatewayRuntimeRegistrar:
    """Class-level registrar for gateway adapters, called by plugins.

    Replaces the hardcoded ADAPTER_REGISTRY dict in GatewayRuntime.
    """
    _adapters: dict[str, type["BaseGatewayAdapter"]] = {}

    @classmethod
    def register_adapter(cls, exchange_type: str, adapter_cls: type) -> None: ...

    @classmethod
    def get_adapter(cls, exchange_type: str) -> type | None: ...

    @classmethod
    def list_adapters(cls) -> list[str]: ...
```

`GatewayRuntime._create_adapter` 改为:

```python
def _create_adapter(self):
    adapter_cls = GatewayRuntimeRegistrar.get_adapter(self.config.exchange_type)
    if adapter_cls is None:
        raise ExchangeNotFoundError(
            f"No adapter registered for {self.config.exchange_type}. "
            f"Install the corresponding plugin package (e.g. bt_api_binance)."
        )
    return adapter_cls(self.config, ...)
```

### 3.5 异常定义

位置:`bt_api_py/plugins/errors.py` (新增)

```python
class PluginError(Exception): ...
class PluginNotFoundError(PluginError): ...
class PluginVersionMismatchError(PluginError): ...
class PluginRegistrationError(PluginError): ...
```

### 3.6 加载器行为规范

`PluginLoader.load_all()` 的契约:

| 场景 | 行为 |
|---|---|
| 插件 entry_point 抛 ImportError | warn log,跳过该插件,继续加载其他 |
| 插件 `register_plugin()` 抛 `PluginRegistrationError` | error log,跳过,继续加载其他 |
| 插件 `register_plugin()` 抛非预期异常 | error log + 完整 traceback,跳过,继续加载其他 |
| 插件返回的 `PluginInfo.core_requires` 与当前核心版本不匹配 | error log,**跳过**该插件,不终止主包启动 |
| 同一 `exchange_type` 被两个插件重复注册 | warn log,保留最先注册者,跳过后注册者 |
| 未找到任何插件 (完全只装了核心) | info log,主包继续正常启动 |

**核心原则:插件故障不得影响主包可用性。**

### 4.2 `bt_api_binance/plugin.py` 样板

```python
from __future__ import annotations

from bt_api_py.balance_utils import simple_balance_handler
from bt_api_py.plugins.protocol import PluginInfo
from bt_api_py.registry import ExchangeRegistry
from bt_api_py.gateway.registrar import GatewayRuntimeRegistrar

from bt_api_binance import __version__
from bt_api_binance.exchange_data.binance import (
    BinanceExchangeDataSpot,
    BinanceExchangeDataSwap,
)
from bt_api_binance.feeds import (
    BinanceAccountWssDataSpot, BinanceAccountWssDataSwap,
    BinanceMarketWssDataSpot, BinanceMarketWssDataSwap,
    BinanceRequestDataSpot, BinanceRequestDataSwap,
)
from bt_api_binance.gateway.adapter import BinanceGatewayAdapter


def _spot_subscribe_handler(data_queue, exchange_params, topics, bt_api):
    # ... 与原 register_binance.py 一致
    ...


def _swap_subscribe_handler(data_queue, exchange_params, topics, bt_api):
    ...


def register_plugin(registry: ExchangeRegistry, runtime_factory) -> PluginInfo:
    # --- ExchangeRegistry 注册 ---
    registry.register_feed("BINANCE___SPOT", BinanceRequestDataSpot)
    registry.register_feed("BINANCE___SWAP", BinanceRequestDataSwap)
    registry.register_exchange_data("BINANCE___SPOT", BinanceExchangeDataSpot)
    registry.register_exchange_data("BINANCE___SWAP", BinanceExchangeDataSwap)
    registry.register_balance_handler("BINANCE___SPOT", simple_balance_handler)
    registry.register_balance_handler("BINANCE___SWAP", simple_balance_handler)
    registry.register_stream("BINANCE___SPOT", "subscribe", _spot_subscribe_handler)
    registry.register_stream("BINANCE___SWAP", "subscribe", _swap_subscribe_handler)

    # --- GatewayRuntime adapter 注册 ---
    GatewayRuntimeRegistrar.register_adapter("BINANCE", BinanceGatewayAdapter)
    # 注: exchange_type 与上面 register_feed 的 key 拆法不同,沿用原来的约定

    return PluginInfo(
        name="bt_api_binance",
        version=__version__,
        core_requires=">=2.0,<3.0",
        supported_exchanges=("BINANCE___SPOT", "BINANCE___SWAP"),
        supported_asset_types=("SPOT", "SWAP"),
    )
```

## 5. 核心包改造清单

按顺序依赖关系列出,必须逐项完成。

### 5.1 改造 1:新增 `bt_api_py/plugins/` 子包

新文件:

```
bt_api_py/plugins/
  __init__.py
  protocol.py       # PluginInfo, 常量定义
  errors.py         # 异常层级
  loader.py         # PluginLoader
```

`loader.py` 核心接口:

```python
class PluginLoader:
    def __init__(self, registry, runtime_registrar):
        self.registry = registry
        self.runtime_registrar = runtime_registrar
        self.loaded: dict[str, PluginInfo] = {}
        self.failed: dict[str, Exception] = {}

    def load_all(self, group: str = "bt_api_py.plugins") -> None:
        """Discover and load all installed plugins."""

    def _load_one(self, ep) -> None: ...
    def _check_core_compatibility(self, info: PluginInfo) -> None: ...
```

### 5.2 改造 2:`bt_api_py/gateway/registrar.py`

见 §3.4。

### 5.3 改造 3:`bt_api_py/gateway/runtime.py`

- **删除** `ADAPTER_REGISTRY` 类属性。
- **删除** `from bt_api_py.gateway.adapters import (Binance|Ctp|IbWeb|Mt5|Okx)GatewayAdapter` 的 top-level import。
- `_create_adapter` 改为通过 `GatewayRuntimeRegistrar.get_adapter()` 查找。
- CTP 的 `OrderRefAllocator` 初始化逻辑**保持原位**,本轮不动 (风险分离)。

### 5.4 改造 4:`bt_api_py/gateway/adapters/__init__.py`

- 当前此文件直接 `from .binance_adapter import BinanceGatewayAdapter`,属于隐性耦合。
- 本轮改为:**继续保留内部 adapter 模块存在**,但 `__init__.py` 不再做顶层 re-export。
- `from bt_api_py.gateway.adapters import BinanceGatewayAdapter` 这种外部调用要给 deprecation warning。
- 真正的 adapter 类由插件包各自持有;主包的 `gateway/adapters/binance_adapter.py` 在 Binance 插件迁移完成后一并删除。

### 5.5 改造 5:`bt_api_py/bt_api.py` 的启动流程

当前:`bt_api.py:78-84` 扫描 `exchange_registers` 主包内部。

本轮改为:

```python
# 1. 先走插件 entry_points
from bt_api_py.plugins.loader import PluginLoader
from bt_api_py.gateway.registrar import GatewayRuntimeRegistrar
_plugin_loader = PluginLoader(ExchangeRegistry, GatewayRuntimeRegistrar)
_plugin_loader.load_all()

# 2. 兼容兜底: 扫描 exchange_registers 主包内部 (带 deprecation warning)
if os.getenv("BT_API_DISABLE_LEGACY_SCAN") != "1":
    _scan_legacy_exchange_registers()
```

- 默认仍然扫描 legacy,保证不破坏现有用户。
- 提供环境变量开关,CI 和试点用户可以显式关闭 legacy,验证纯插件路径。

### 5.6 改造 6:`bt_api_py/feeds/registry.py:142-151`

- 当前硬编码 `import bt_api_py.exchange_registers.register_binance` 等三个。
- 本轮改为:用 try/except 逐个包裹,避免迁移 Binance 时核心包 ImportError;长期应整体移除。

## 7. CI 静态检查设计

### 7.1 "核心包禁止 import 插件" 静态检查

新增脚本:`scripts/check_core_isolation.py`

```python
#!/usr/bin/env python
"""Fail CI if bt_api_py core imports any bt_api_<exchange> plugin."""
import pathlib, re, sys

FORBIDDEN = re.compile(
    r"^\s*(from|import)\s+bt_api_(binance|okx|ctp|ib_web|mt5)\b",
    re.MULTILINE,
)
CORE_DIR = pathlib.Path("bt_api_py")

failures = []
for py in CORE_DIR.rglob("*.py"):
    text = py.read_text(encoding="utf-8")
    for m in FORBIDDEN.finditer(text):
        failures.append(f"{py}:{text[:m.start()].count(chr(10))+1}: {m.group().strip()}")

if failures:
    print("Core package must not import plugin packages:")
    for f in failures:
        print("  " + f)
    sys.exit(1)
```

## 8.5 硬性约束:测试必须跟随被测代码迁移 🔒

本项**不是建议,是契约**,适用于迭代 001 及后续所有拆包波次(Wave 2~7)。

### 8.5.1 约束内容

当某个交易所的生产代码被从主仓迁出或删除时,**其对应的测试代码必须同步迁出或同步删除**,主仓 `tests/` 目录不得保留:

1. 与被迁/被删交易所相关的任何 `test_<exchange>_*.py` 文件
2. 与被迁/被删交易所相关的任何 `tests/feeds/live_<exchange>/` 目录
3. 与被迁/被删交易所相关的任何 `tests/integration/*<exchange>*` 文件
4. 与被迁/被删交易所相关的任何 conftest.py 中的 exchange 专用 fixture(应迁到插件包内)

### 8.5.2 迁移方向对照表

| 主仓源路径 | 目标 (拆包场景) | 目标 (删除场景) |
|---|---|---|
| `tests/test_<x>_balance.py` | `packages/bt_api_<x>/tests/test_balance.py` | `git rm` |
| `tests/test_<x>_exchange_data.py` | `packages/bt_api_<x>/tests/test_exchange_data.py` | `git rm` |
| `tests/feeds/live_<x>/` (整目录) | `packages/bt_api_<x>/tests/feeds/` | `git rm -r` |
| `tests/integration/*<x>*` | `packages/bt_api_<x>/tests/integration/` | `git rm` |
| `tests/conftest.py` 中 `<x>_*` fixture | 移到 `packages/bt_api_<x>/tests/conftest.py` | 删除该 fixture |

### 8.5.3 PR 合并门禁

包含交易所迁移 / 删除的 PR,合并前必须通过以下自动化检查(在 CI 增加一个 job):

```bash
# 假设被迁/删的交易所名为 $X
EXCHANGE=$1

# 1. 主仓 tests/ 下不得残留任何 $X 相关文件
if find tests/ -iname "*${EXCHANGE}*" -type f | grep -q .; then
    echo "FAIL: tests/ still contains ${EXCHANGE} files"
    exit 1
fi

# 2. 主仓 tests/ 下不得 import bt_api_py.feeds.live_<x>
if grep -rnE "bt_api_py\.feeds\.live_${EXCHANGE}\b" tests/; then
    echo "FAIL: tests/ still imports live_${EXCHANGE}"
    exit 1
fi

# 3. pytest collect 不得发现 $X 相关用例
if pytest tests/ -k "${EXCHANGE}" --collect-only 2>&1 | grep -q "collected [1-9]"; then
    echo "FAIL: pytest still collects ${EXCHANGE} tests"
    exit 1
fi

echo "PASS: ${EXCHANGE} fully cleaned from main tests/"
```

### 8.5.4 插件包测试的独立性

每个 `packages/bt_api_<x>/` 必须满足:

1. `cd packages/bt_api_<x> && pip install -e . && pytest tests/ -v` 可独立跑通
2. 不反向 import 主仓的任何 `tests/*` 模块
3. 共享 fixture 通过 `bt_api_py.testing`(迭代 001 的 T108 产出)提供
4. 集成测试的环境变量(API key 等)在插件包自己的 README 和 CI 文档里说明

### 8.5.5 违反后果

- Code review 阶段发现未同步迁/删测试:**直接拒审**,要求补交 commit
- CI 的 `core-tests-cleanup` job 失败:**阻断合并**
- 合并后发现残留:**立即 revert**,不做"下一个 PR 补"
