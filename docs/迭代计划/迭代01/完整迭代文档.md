# 迭代01：插件化架构重构

> 版本：v1.0
> 日期：2026-04-17
> 状态：规划中

---

## 1. 背景与目标

### 1.1 当前问题

`bt_api_py` 已经相对成熟，接入了 73+ 个交易所，但随着功能不断堆积，整个项目变得臃肿：

- **体积庞大**：用户只需要其中 2-3 个交易所，却要安装整个仓库
- **依赖复杂**：CTP 需要 C++ 编译，IB Web 需要 Playwright，MT5 需要本地终端
- **发布困难**：任何一个交易所的改动都需要全量测试和发布
- **协作困难**：多人同时修改主仓库的冲突频繁

### 1.2 目标

将交易所相关代码拆分出主仓库，形成独立的插件包：

| 包名 | 说明 | 依赖特殊性 |
|------|------|------------|
| `bt_api_base` | 基础层，所有包都依赖它 | 无 |
| `bt_api_py` | 核心包，统一接口 + gateway + zmq | 依赖 bt_api_base |
| `bt_api_ctp` | 中国期货 | C++ SWIG 扩展 |
| `bt_api_ib_web` | Interactive Brokers Web | Playwright 浏览器自动化 |
| `bt_api_mt5` | MetaTrader 5 | 本地 MT5 终端 |
| `bt_api_binance` | 币安 | 无特殊依赖 |
| `bt_api_okx` | OKX | 无特殊依赖 |

### 1.3 架构设计总图

```
┌──────────────────────────────────────────────────────────────┐
│                         用户代码                               │
│              BtApi({"exchange": "BINANCE___SPOT"})           │
└─────────────────────────┬────────────────────────────────────┘
                          │
                          ▼
┌──────────────────────────────────────────────────────────────┐
│                    bt_api_py (核心包)                          │
│  ├── PluginLoader        ← 插件发现与加载                      │
│  ├── ExchangeRegistry    ← 交易所注册中心                      │
│  ├── GatewayRuntime      ← 统一网关（含 zmq 转发）            │
│  └── EventBus           ← 事件总线                            │
│                                                              │
│  依赖: bt_api_base (仅此一项)                                 │
└─────────────────────────┬──────────────────────────────────────┘
                          │
          ┌───────────────┼───────────────┐
          │               │               │
          ▼               ▼               ▼
┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐
│  bt_api_binance │ │    bt_api_okx   │ │    bt_api_ctp   │
│  (独立包)       │ │    (独立包)      │ │    (独立包)      │
│                 │ │                 │ │                 │
│  ├── feeds/     │ │  ├── feeds/     │ │  ├── feeds/     │
│  ├── gateway/   │ │  ├── gateway/   │ │  ├── gateway/   │
│  └── exchange_  │ │  └── exchange_  │ │  └── ctp/      │
│      data/      │ │      data/       │ │      (C++扩展)  │
└────────┬────────┘ └────────┬────────┘ └────────┬────────┘
         │                  │                  │
         └──────────────────┼──────────────────┘
                            │
                            ▼
              ┌─────────────────────────────┐
              │       bt_api_base (基础包)    │
              │                             │
              │  ├── containers/ (数据容器)   │
              │  ├── exceptions/ (异常定义)   │
              │  ├── balance_utils/ (余额处理) │
              │  ├── rate_limiter/ (限流器)   │
              │  └── testing/ (测试工具)      │
              └─────────────────────────────┘
```

### 1.4 核心原则

1. **bt_api_xx 只能依赖 bt_api_base**：插件包之间不得相互依赖
2. **bt_api_py 依赖 bt_api_base**：核心包通过插件间接访问交易所
3. **bt_api_base 无交易所逻辑**：只包含所有插件都会用到的通用代码
4. **插件独立发布**：每个插件可单独 pip install 和发布
5. **测试必须跟随代码迁移**：插件的测试必须在插件包内，不得留在主仓库

## 3. 插件契约规范

### 3.1 entry_points group

```toml
# 插件包 pyproject.toml
[project.entry-points."bt_api_py.plugins"]
binance = "bt_api_binance.plugin:register_plugin"
```

### 3.2 PluginInfo 数据类

```python
@dataclass(frozen=True)
class PluginInfo:
    name: str                      # 如 "bt_api_binance"
    version: str                    # 遵循 semver
    core_requires: str              # PEP 440 规格，如 ">=2.0,<3.0"
    supported_exchanges: tuple[str, ...]  # 如 ("BINANCE___SPOT",)
    supported_asset_types: tuple[str, ...] # 如 ("SPOT", "SWAP")
```

### 3.3 插件入口函数

```python
def register_plugin(
    registry: "ExchangeRegistry",
    runtime_factory: "GatewayRuntimeRegistrar",
) -> PluginInfo:
    """注册插件能力，被核心在启动时调用一次。"""
```

## 5. 验收指标

### 5.1 核心包验收

- [ ] `pip install bt_api_py` 不含任何交易所代码
- [ ] `import bt_api_py` 不报 ImportError
- [ ] `BtApi()` 可构造（无任何插件时）
- [ ] 核心包内代码不得 import 任何 `bt_api_xx` 插件包

### 5.2 插件包验收

- [ ] `pip install bt_api_binance` 后，Binance 交易所可用
- [ ] `pip install bt_api_ctp` 后，CTP 交易所可用
- [ ] 各插件独立运行 `pytest tests/` 全绿

### 5.3 性能指标

- `import bt_api_py` 耗时不超过基线 1.2 倍
- `BtApi(...)` 构造耗时不超过基线 1.2 倍

## 6. 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| CTP C++ 扩展跨平台编译 | 高 | 提前 PoC，CI 矩阵覆盖三平台 |
| 插件版本与核心版本不兼容 | 中 | 严格版本检查，失败隔离 |
| 用户迁移路径不清 | 中 | 完整迁移文档 + deprecation warning |
| 循环依赖 | 高 | bt_api_base 先做，设计阶段审查依赖图 |
