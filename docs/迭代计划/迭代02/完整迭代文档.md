## 3. ZMQ 分发设计（核心新增功能）

### 3.1 消息协议

```
Topic 格式:  {exchange}.{asset_type}.{symbol}.{data_type}
示例:        BINANCE.SPOT.BTC-USDT.ticker
             OKX.SWAP.ETH-USDT.depth
             BINANCE.SWAP.BTC-USDT.order
             BINANCE.SWAP.BTC-USDT.trade

消息体:  msgpack 序列化的 dict
{
    "topic": "BINANCE.SPOT.BTC-USDT.ticker",
    "timestamp": 1713600000000,  // 毫秒时间戳
    "data": { ... },             // 具体数据，结构取决于 data_type
    "sequence": 12345            // 序列号，用于检测丢包
}
```

### 3.2 通信模式

```
行情分发（PUB/SUB）:
┌──────────┐    PUB     ┌──────────┐
│ BtApi    │ ──────────▶│ Strategy1│ SUB: BINANCE.SPOT.*.ticker
│ ZMQ PUB  │ ──────────▶│ Strategy2│ SUB: *.*.BTC-USDT.*
│ :5555    │ ──────────▶│ Strategy3│ SUB: OKX.SWAP.*.*
└──────────┘            └──────────┘

交易指令（REQ/REP）:
┌──────────┐    REQ     ┌──────────┐
│ Strategy │ ──────────▶│ BtApi    │
│ ZMQ REQ  │ ◀──────────│ ZMQ REP  │
│          │    REP     │ :5556    │
└──────────┘            └──────────┘

请求格式:
{
    "action": "make_order",
    "request_id": "uuid",
    "params": {
        "exchange": "BINANCE___SWAP",
        "symbol": "BTC-USDT",
        "side": "buy",
        "price": 50000,
        "volume": 0.01
    }
}

响应格式:
{
    "request_id": "uuid",
    "success": true,
    "data": { ... },
    "error": null
}
```

### 3.3 模块结构

```
bt_api_py/zmq/
├── __init__.py
├── config.py           # ZMQ 配置（端口、序列化方式、心跳间隔等）
├── publisher.py        # ZmqPublisher - 行情推送
├── subscriber.py       # ZmqSubscriber - 行情订阅客户端
├── request_reply.py    # ZmqRequestReply - 交易指令 REQ/REP
├── serializer.py       # 消息序列化（msgpack/json 可切换）
├── monitor.py          # 连接监控与统计
└── protocol.py         # 消息协议定义（Topic、Message dataclass）
```

## 4. DirectClient 标准接口

### 4.1 AbstractDirectClient

```python
class AbstractDirectClient:
    """每个 bt_api_xx 必须实现的直连客户端接口"""

    # 行情
    async def get_ticker(self, symbol: str) -> Ticker: ...
    async def get_depth(self, symbol: str, limit: int = 20) -> OrderBook: ...
    async def get_kline(self, symbol: str, period: str, limit: int = 100) -> list[Bar]: ...
    async def get_trades(self, symbol: str, limit: int = 100) -> list[Trade]: ...

    # 交易
    async def place_order(self, symbol: str, side: str, order_type: str,
                          price: float, quantity: float, **kwargs) -> Order: ...
    async def cancel_order(self, symbol: str, order_id: str) -> bool: ...
    async def get_order(self, symbol: str, order_id: str) -> Order: ...
    async def get_open_orders(self, symbol: str | None = None) -> list[Order]: ...

    # 账户
    async def get_balance(self) -> list[Balance]: ...
    async def get_positions(self, symbol: str | None = None) -> list[Position]: ...

    # WebSocket
    async def subscribe_ticker(self, symbol: str, callback: Callable) -> str: ...
    async def subscribe_depth(self, symbol: str, callback: Callable) -> str: ...
    async def unsubscribe(self, subscription_id: str) -> bool: ...

    # 生命周期
    async def connect(self) -> None: ...
    async def disconnect(self) -> None: ...
    async def __aenter__(self): ...
    async def __aexit__(self, *args): ...
```

### 4.2 使用方式

```python
# 方式A：直连（仅需 bt_api_base + bt_api_binance）
from bt_api_binance import BinanceDirectClient

async with BinanceDirectClient(api_key="...", secret="...") as client:
    ticker = await client.get_ticker("BTC-USDT")
    order = await client.place_order("BTC-USDT", "buy", "limit", 50000, 0.01)

# 方式B：统一接口（需要 bt_api_py + bt_api_binance）
from bt_api_py import BtApi

api = BtApi({"BINANCE___SPOT": {"api_key": "...", "secret": "..."}})
ticker = api.get_tick("BINANCE___SPOT___BTC-USDT")

# 方式C：ZMQ 分发（需要 bt_api_py + bt_api_binance）
from bt_api_py.zmq import ZmqSubscriber, ZmqTradeClient

# 行情订阅
sub = ZmqSubscriber("tcp://localhost:5555")
sub.subscribe("BINANCE.SPOT.BTC-USDT.ticker", callback=on_ticker)

# 交易指令
trade_client = ZmqTradeClient("tcp://localhost:5556")
result = await trade_client.place_order(exchange="BINANCE___SPOT", symbol="BTC-USDT", ...)
```

## 7. 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| ZMQ 性能瓶颈（高频行情场景） | 高 | 使用 msgpack 序列化 + inproc transport 测试基线性能 |
| exchange_registers 删除后向后兼容破坏 | 中 | 提供明确的 deprecation warning 和迁移文档 |
| DirectClient 接口过于理想化 | 中 | 先实现 binance 作为模板，再根据实际调整接口 |
| 各包版本管理混乱 | 中 | 使用 monorepo 工具统一版本，或约定版本同步策略 |
| CTP DirectClient 跨平台兼容 | 高 | 限定 Linux 为主要目标平台，macOS/Windows 为 best-effort |
